03 Reflection & Scope

Key Advantages

Eliminates roommate disputes prior to allocation. Standardized objective scoring. Resilient cloud backend architecture with LocalStorage fallback.

System Limitations

Relies on honest, accurate student survey responses. Clever Cloud free tier database caps active SQL request concurrency.

Future Scope

Clustering Machine Learning models (K-Means) for multi-bed configurations. Anonymous secure peer chat verification before room finalization.

System Integration

Built with scalable API architecture for seamless onboarding of institutional management portals and student databases.

04 Weighted Matching Index
25% Sleep Schedule
25% Cleanliness
20% Study Habits
30% Hobbies & Social

Weight Allocation: Lifestyle habits are weighted based on administrator-configured priorities. The compatibility index aggregates these components using a normalized linear combination to yield a final matching score.

05 Jaccard Similarity Index
J(A, B) = |A ∩ B| / |A ∪ B|

Interest overlaps are calculated by dividing the shared hobbies (intersection) by the total combined hobbies (union) between Roommate A and Roommate B.

Set Logic: The set intersection \(|A \cap B|\) extracts hobbies shared in common, while the union \(|A \cup B|\) calculates total unique hobbies. The resulting ratio represents set similarity, acting as a primary filter for interest-based matches.
